轻量化心理陪伴 AI Agent 设计与实现 —— 知我心理健康平台开发随笔
1. Agent 说明文档
1.1 需求描述
本项目为知我匿名心理健康 Web 平台配套 AI Agent 中间服务，核心解决前端直接暴露大模型 API 密钥、单一模型接口故障导致对话功能失效、AI 无法联动心理量表评测三大痛点。
隐私安全需求：隔离前端与大模型原始密钥，密钥仅存储于服务端 Agent 层，前端无敏感凭证；
高可用需求：实现 DeepSeek、Coze 双模型路由容灾，主模型调用失败自动切换备用接口；
业务联动需求：识别用户倾诉内容中的负面情绪，输出标记指令引导前端跳转 PHQ-9/GAD-7 心理量表测评；
交互体验需求：基于 SSE 流式长连接，逐字推送 AI 回复，模拟真人打字机对话效果；
轻量化适配需求：适配无后端纯前端 Web 项目，无需数据库，仅做请求转发与内容解析。
1.2 业务流描述
用户在前端对话页面输入情绪倾诉文本，携带会话 ID 发起请求至 Agent 服务；
Agent 接收请求，优先调用主模型 DeepSeek 接口，传入心理疏导限定 Prompt；
模型返回流式文本，Agent 实时转发至前端，逐行推送实现打字机效果；
若 AI 回复识别到用户存在明显焦虑、抑郁情绪，输出[ASSESSMENT_COMPLETE]标记；
前端捕获标记后，自动弹出量表测评引导弹窗，跳转评测页面；
若主模型接口超时 / 额度耗尽 / 报错，Agent 自动切换至 Coze 备用模型，无缝续接对话；
会话结束后 Agent 销毁临时会话缓存，不存储用户对话内容，保障隐私。
完整流程链路：
用户输入 → 前端请求 → Agent 路由分发 → 大模型流式返回 → Agent 解析内容转发前端 → 识别情绪标记 → 前端跳转量表 / 持续对话
1.3 实现说明
分层架构设计
前端交互层：原生 JS 封装 SSE 请求，监听 Agent 推送流，渲染打字机对话；
Agent 中转层：TypeScript 实现，核心包含路由分发模块、异常捕获模块、标记解析模块；
大模型对接层：统一封装两套模型请求格式，标准化入参、出参结构。
容灾逻辑实现
设置接口超时阈值 3s，主模型请求抛出任意异常时，立即复用当前用户对话上下文，发起备用模型请求，不中断用户对话。
情绪联动逻辑
固定系统 Prompt 约束 AI：当用户持续表达低落、失眠、自我否定、焦虑等情绪时，回复末尾附加固定标记[ASSESSMENT_COMPLETE]，Agent 完整透传标记至前端。
安全实现
所有 API 密钥存放服务端环境变量，Agent 仅转发用户纯文本内容，不向前端返回任何密钥、接口地址；无用户身份采集，会话 ID 仅临时存储于浏览器 sessionStorage。
流式通信实现
采用 Server-Sent Events 单向长连接，Agent 分段接收模型输出，逐块推送至前端，降低一次性加载延迟。
1.4 使用示例（功能演示）
示例 1：普通情绪倾诉对话
用户输入：最近学业压力很大，每天都睡不着，情绪很低落；
Agent 转发至 DeepSeek 模型，AI 分段流式输出安慰文字；
AI 识别负面情绪，回复末尾携带[ASSESSMENT_COMPLETE]；
前端捕获标记，弹窗提示：“检测到你存在情绪困扰，建议完成专业心理自评量表”，提供跳转按钮。
示例 2：主模型故障自动切换
DeepSeek 接口额度耗尽返回 403 错误；
Agent 捕获异常，自动携带历史对话上下文调用 Coze 模型；
对话无中断，继续回复用户，前端无报错提示。
示例 3：无负面情绪日常倾诉
用户输入：今天出门散步心情好了很多；AI 正常回复疏导内容，不携带测评标记，页面持续对话模式。
