一、学期回顾
1.1 回顾你对于软件工程课程的想象
开课之前，我以为软件工程只是单纯学习写代码、做程序开发，期待能掌握实用的编程技巧做出完整项目。经过一学期学习后，达成期待的部分：
掌握了完整软件项目从需求分析、方案设计、编码实现到测试交付的全流程规范，学会用专业文档梳理项目逻辑，不再是想到哪写到哪的粗放开发；
理解了团队协作开发的标准流程，学会用 Git 做版本管理、GitHub 托管项目代码，实现多人并行开发不冲突；
能独立完成 AI Agent 这类轻量化中间服务的设计，把课堂上学的架构分层、容灾设计知识用到了实际项目里。
仍存在不足的部分及原因：
项目测试环节做得不够细致，只完成了基础功能测试，没有覆盖边界场景、性能压力测试，原因是前期把大部分时间投入了功能开发，对软件测试的流程规范学习深度不足；
项目进度预判能力不足，两次临近作业截止才出现小范围赶工情况，原因是前期对任务拆分颗粒度太粗，没有给每个子任务预留缓冲调整时间；
文档迭代的及时性不足，代码更新后偶尔会滞后更新设计文档，原因是一开始没养成 “文档和代码同步维护” 的习惯。
1.2 回顾你在这门课程中的投入与产出
在软工实践课程当中我累计编写了约 1200 行代码，涵盖 AI Agent 服务 TypeScript 代码、前端页面 HTML/CSS/JS 代码、量表计分逻辑脚本。
在团队项目中我参与了「知我心理健康 AI 陪伴平台」的设计与开发，承担 PM + 文档负责人的角色，负责需求拆解、全套设计文档撰写、项目进度统筹、答辩材料制作与团队沟通协调。
软工实践各次作业花费时间：
第一次团队作业（搭建 GitHub 仓库 + 团队分工）：6 小时
第二次团队作业（需求分析文档编写）：10 小时
第一次团队项目作业（系统 + 数据库设计）：12 小时
第二次团队项目作业（原型设计 + 前端初稿开发）：15 小时
第三次团队项目作业（AI Agent 开发 + 前后端联调）：18 小时
第四次团队项目作业（全项目测试 + 随笔博客 + 答辩准备）：14 小时
在软件工程课程上花费的时间：
累计总时长 75 小时，实际每周平均投入 4.7 小时，课程初期预估每周投入 4 小时。
1.3 令你印象最深刻的一次作业 / 答辩
印象最深的是第三次团队项目 AI Agent 开发作业。
原因有两点：第一，这是项目技术难点最集中的环节，要实现双大模型容灾调度、SSE 流式通信、情绪联动量表跳转逻辑，前期反复调试接口异常、长连接断连的问题，和队友同步调整了 3 版方案才跑通稳定效果，攻克难点的成就感很强；第二，这次作业真正让我体会到软件工程 “分层架构、提前规避风险” 的思想，Agent 中转层既解决了前端密钥泄露的安全隐患，又做了故障容灾，把课堂理论完全落地，也让我理解了软件设计前置的重要性。
二、总结收获
2.1 展开说说你的软工实践故事
前期团队搭建阶段
第一次团队作业我们两人组队，确定团队名称与项目方向，从零搭建公共 GitHub 仓库，梳理每个人的固定分工。最开始我们不熟悉 Git 上传流程，文件上传出现重复覆盖、版本混乱的问题，反复查阅教程后掌握网页上传、命令行提交两种方式，也定下每次修改前同步沟通的规则，为后续协作打下基础。这次经历让我明白版本管理是多人协作的基础，规范的仓库目录能大幅降低后期文件查找成本。
需求与设计文档阶段
完成需求、系统、数据库三份设计文档是第二、三次作业核心任务。初期我们绘制的 UML 图、ER 实体关系存在多处逻辑冲突，用例描述和页面功能不匹配，我作为文档负责人，需要逐条对照开发思路修正文字与图表，每天和队友线上核对数据存储、页面流转逻辑。举例来说，最初文档里 sessionStorage 存储字段定义和前端规划不一致，我们同步修改两处内容，避免后续开发出现数据传参 bug，也总结出 “文档和开发同步更新” 的协作准则。
冲刺开发与 AI Agent 落地阶段
集中冲刺阶段我们分工完成前端页面与 AI 中间服务开发，本次印象最深的 Agent 模块出现接口不稳定问题，主模型调用时常报错中断对话。我和队友共同梳理解决方案，设计双模型自动切换的容灾逻辑，同时通过固定 Prompt 约束 AI 识别负面情绪，输出标记联动量表测评。多次调试后功能完整跑通，这个过程让我学会拆解复杂技术问题，分模块排查故障，不再单一等待问题自行解决。
交付收尾与博客随笔阶段
最后一次作业需要撰写 Markdown 博客随笔、整理全部交付物、完善仓库内容。我独立完成 Agent 说明文档、小组分工、成员心得的文字撰写，同步整理三份设计文档、源码压缩包、团队介绍文件统一上传 GitHub，梳理仓库 README 介绍项目结构。完整走完从需求到交付的全流程后，我直观感受到软件工程规范化交付的完整标准。
2.2 介绍学习到的新技术或生产力工具以及它们带来的帮助
Git & GitHub：掌握代码仓库托管、版本提交、文件上传操作，实现多人项目文件统一管理，随时回溯历史版本，同时可对外分享项目链接完成课程交付，解决团队文件传输混乱的问题。
Markdown 语法：用于撰写课程博客随笔，轻量化排版，结构清晰，适配博客园、GitHub 文档阅读场景，简化长文本排版工作。
SSE 流式通信技术：实现 AI 对话打字机实时输出效果，理解前后端长连接交互逻辑，掌握大模型对话场景的交互开发方案。
TypeScript：用于编写 AI Agent 中转服务，静态类型检查减少接口传参错误，分层拆分代码提升程序可读性，便于后续维护迭代。
UML 绘图工具：绘制用例图、时序图、ER 实体关系图，标准化梳理项目业务流程，让抽象的产品逻辑可视化，便于团队统一理解需求。
2.3 技术之外，这门课程还带来的提升
项目统筹规划能力：学会拆分大型项目任务，制定阶段性冲刺计划，合理分配两人工作量，把控作业交付时间节点，规避赶工、遗漏交付物的问题。
高效团队沟通能力：遇到文档逻辑、代码功能分歧时，能够清晰表达自身思路，协同商议折中解决方案，学会站对方视角调整工作内容，减少协作矛盾。
问题拆解与抗压心态：面对复杂技术难题、大量文档撰写任务时，不再产生畏难情绪，能够拆分小任务逐个解决，适应多任务并行的学习节奏。
标准化书面表达能力：能够按照软件工程规范撰写专业设计文档，逻辑严谨、结构清晰，提升书面总结、技术说明的文字输出能力。
复盘自省意识：每次作业完成后主动梳理项目短板，记录开发、文档中的漏洞，总结优化方案，养成做完项目复盘的习惯。
2.4 自由记录与感想
这门课彻底改变了我对软件开发的认知，从前我认为编程只是独立写代码，现在明白任何可用的软件都离不开规范设计、团队协作、完整交付流程，工程化思维比单纯写代码更加重要。本次开发的心理自评平台贴合青年群体真实需求，让我体会到技术可以落地解决现实生活里的痛点，也让我对前端 + AI 结合的开发方向产生浓厚兴趣，未来会继续深耕相关技术。
本次实践最大的遗憾是前期没有预留充足测试时间，项目缺少完善的边界测试用例，后续自主开发项目时我会提前规划测试环节。
想送给学弟学妹一句建议：不要等到截止日前才动手完成软工作业，提前拆分任务、同步沟通文档与代码，循序渐进完成项目，会大幅减少开发与修改的压力，也能真正吃透课程里的工程化知识。

这段软工实践旅程已经结束，学到的工程思维、协作方法都会成为后续专业学习的重要基础。
